# ケーススタディ：倉庫管理システム移行（仮想プロジェクト）

---

## 1. 本ケーススタディの目的

本ドキュメントは、  
I/O境界を基軸とした移行スコープ設計モデルを  
中規模レガシー倉庫管理システムに適用した仮想ケースです。

目的は以下の設計能力を明示することです。

- I/O境界による移行スコープ定義
- データ境界（SoR / Write Owner）による依存制御
- ハイブリッド移行戦略の判断
- 段階移行を成立させる設計条件の明文化
- 移行リスクの構造的整理

本ケースは実案件ではなく、設計能力提示を目的とした仮想適用例です。

---

## 2. システム概要（仮想設定）

### 対象システム
製造業向け 倉庫管理システム（WMS）

### 現行構成
- COBOLプログラム：55本  
  - バッチ：40本  
  - オンライン：15本  
- 画面：10
- 外部IF：6
- 帳票：8
- データ構成：
  - VSAM（業務中核データ）
  - RDB（参照／ログ／集計）

### 移行目的
- 業務停止を伴わない段階移行
- レガシー依存の段階的縮小
- 状態遷移ロジックの明確化
- 将来拡張可能な構造への転換

---

## 3. I/O境界によるスコープ定義

入出力単位で業務を再分割し、以下の6スコープを定義した。

1. 入荷処理（Inbound）
2. 出荷処理（Outbound）
3. 在庫参照
4. 在庫統制（棚卸）
5. マスタ同期
6. 外部IFハブ

設計方針：

- プログラム単位ではなく I/O単位で境界を定義
- 状態遷移を持つスコープを明確化
- 参照専用スコープは独立させる

---

## 4. データ境界設計（SoR / Write Owner）

### 中核データ領域

| データ領域 | 格納 | SoR | 更新責任スコープ |
|------------|------|------|------------------|
| 入出荷 | VSAM | WMS | S1 / S2 |
| 在庫 | VSAM | WMS | S1 / S2 / S4 |
| ロケーション | VSAM | WMS | S1 / S2 / S4 |
| 商品マスタ | RDB | ERP | S5 |
| 作業実績 | RDB | WMS | S1 / S2 / S4 |

### 設計原則

1. 更新責任は一意に定義する
2. 外部IFは状態遷移を持たない
3. 在庫SoRは最後に移行する
4. マスタは独立移行可能とする

---

## 5. 移行方式の選定（ハイブリッド戦略）

### 採用方式

周辺ラップ＋中核リライト

### 判断基準

| スコープ | 方針 | 判断理由 |
|------------|--------|-----------|
| マスタ同期 | 先行リライト | 依存度低・早期価値創出 |
| 在庫参照 | 早期リライト | 参照専用で安全 |
| 外部IFハブ | 先行リライト | 境界統制点を確立 |
| 入荷 | 段階的リライト | 状態遷移中核 |
| 出荷 | 段階的リライト | 業務中心フロー |
| 在庫SoR | 最終移行 | 高リスク領域 |
| ロケーション | 在庫後 | 強結合領域 |

---

## 6. 段階移行計画

### Phase 0：境界整備
- 外部IFハブ構築
- マスタ同期構築
- 在庫参照系の新設

目的：新旧共存の統制点確立

---

### Phase 1：状態モデル固定
- 入出荷状態遷移の明文化
- 旧系との比較検証

目的：中核ロジックの安定化

---

### Phase 2：入出荷リライト
- 状態遷移ロジックを新系へ移行
- 在庫SoRは旧系維持

目的：中核制御の移管

---

### Phase 3：在庫SoR移行
- 更新責任を新系へ統合
- 棚卸制御統合

目的：SoRの切替完了

---

### Phase 4：ロケーション統合
- 強結合領域の最終移行
- 旧系廃止

---

## 7. リスク整理

| リスク | 設計上の対策 |
|--------|----------------|
| 在庫二重更新 | 更新責任固定 |
| 状態遷移不整合 | 状態遷移責任点の明確化 |
| IF順序問題 | 冪等処理＋キュー制御 |
| 棚卸補正誤適用 | 統制スコープ限定 |

---

## 8. 本ケースの意義

本ケースは、

- 移行境界の定義
- データ所有権の明確化
- 段階移行成立条件の提示
- リスク構造の可視化

を示すための設計モデル適用例である。

目的は実装提示ではなく、

「移行を成立させる設計条件を定義できること」

の明示にある。

# English Summary

This virtual case study demonstrates how an I/O-boundary-based migration scope model can be applied to a mid-scale warehouse management system.

The design focuses on:

- I/O-based scope definition
- Data ownership (SoR / Write owner) control
- Hybrid migration strategy (wrap peripheral / rewrite core)
- Phased migration with dependency control
- Risk-based migration decision-making

The purpose is not to present implementation details, but to demonstrate the ability to define conditions under which migration can safely proceed.
